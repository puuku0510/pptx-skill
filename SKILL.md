# PPTX スライド自動生成スキル v3

## 概要
台本（スプレッドシート or CSV or MD）を受け取り、PptxGenJSでGoogleスライド互換のPPTXファイルを**一発生成**する。
2パターンに対応。パターンと枚数はユーザーが指定する。

> [!CAUTION]
> **1台本 ＝ 1パターンルール**
> 1つの台本で複数パターンを混在させない。
> 混ぜたい場合は台本を分割して2回生成→結合する。

## ワークフロー（5ステップ）— 厳守

> [!CAUTION]
> 台本を渡されたら**必ずヒアリング→構成案→承認**を経てから制作に入ること。
> 台本を受け取って即制作はNG。**フローを絶対に飛ばさない。**

### ① トリガー
ディレクターが「スライド作って」等で起動。

### ② ヒアリング（必須）
Antigravityが以下を**必ず確認**してから進める：
```
以下を教えてください：
1. 台本（スプシURL / CSVパス / テキスト貼り付け）
2. パターン: source / character（省略時はチャンネルのデフォルト）
3. 枚数: 何枚？（10〜50枚）
4. チャンネル名（下のプロファイル参照）
5. 素材パス（デフォルト: C:\KEI IWASAKI\assets\{チャンネル名}\ でOK？）
6. 特別な指示（任意）
```

省略された項目はデフォルト値を使用：
- パターン: `source`
- 枚数: `30`（レンジ: 10〜50枚）
- ファイル名: `output/slides.pptx`

### ③ 構成案を提示
Antigravityが台本を読み込み、**1行1スライドのリスト**を提示：
```
構成案（35枚）：
#01 [text] 配当投資は上位7%の勝ち組
#02 [table] 不労所得の比較表（○△×）
#03 [formula] NISA枠の計算
...
#35 [cta] LINE登録CTA
```
> 5秒で確認できる粒度。各行にスライドタイプを明記。

### ④ 承認
ディレクター: 「OK」or「#03を分割して」「#15にソース追加」等の修正指示。
修正があれば構成案を更新して再提示。

### ⑤ 制作
承認後、一括生成 → `output/` にPPTX保存 → **PPTX + PDF両方**を納品。

> [!IMPORTANT]
> ③の構成案確認がある分、**完成品の修正工数が激減**する。

> [!IMPORTANT]
> **納品時はPPTXとPDFの両方を必ず送ること！**
> スマホではPPTXの画像が表示されない場合がある。
> PDF変換: `soffice --headless --convert-to pdf output.pptx`（LibreOffice推奨）
> フォールバック: PowerPoint COM `SaveAs($pdf, 32)`

> [!WARNING]
> **Discord納品フォールバックルール**
> - ファイルサイズ < 8MB → Discord直接添付
> - ファイルサイズ ≥ 8MB → `shared-context/` に保存 + Discordにファイルパスだけ通知
> - Discord送信は `Invoke-RestMethod` を使用（PowerShellの`curl`エイリアスは使わない）

## 2つのパターン

### パターン1: `source`（図解特化テロップ）
- **画風**: 赤帯ヘッダー＋**白背景**
- **カラー**: **赤ベース**（headerRed/textRed/highlight黄色）
- **特徴**: **図解・表・グラフ・数式に特化**。テキストのみのスライドは原則禁止
- **フォント**: 最小20pt、見出し28pt+、キーワード36pt+、全太字
- **出典**: 統一書式 `出典：◯◯ (YYYY/MM/DD)` 20pt

> [!CAUTION]
> **パターン1 情報密度ルール（v4 厳守）**
> - **テキストのみスライド禁止**: 必ず表・グラフ・図解・数式・タイムラインのいずれかを含めること
> - **1スライド = 1つの「見れば分かる」ビジュアル**: テキストの読み上げではなく、視覚で理解できるものを作る
> - **参考基準**: `C:\KEI IWASAKI\参考スライド\パターン1\` の画像密度を基準
> - **画像収集は別スキル**: スクショ・引用画像の収集は `slide-source-collector` スキルで行う。パターン1ではスクショ貼り付けスライドは作らない
> - **枚数は絞る**: 台本全体で10〜20枚が目安。多くなくてよい。質を優先
> - **情報密度**: 1枚に複数の数値・比較・ステップを詰め込む（参考スライドの密度が最低ライン）

### パターン2: `character`（キャラ系テロップ）
- **画風**: キャラ＋吹き出し＋太字影テキスト＋コラージュ配置
- **カラー**: **赤ベースに統一**（赤/オレンジ/ピンクのバリエーション）
- **素材**: ローカルフォルダから毎回読み込み
- **特徴**: ムード判定でキャラ自動選択、レイアウトローテーション

> [!IMPORTANT]
> **パターン2 デザインルール（厳守）**
> 1. **画像アスペクト比固定** — `addImageExact()` 使用（image-sizeで実寸計算→100%歪みなし）
> 2. **最小フォント20pt** — パターン1と同ルール
> 3. **賑やかさ** — 1スライド4〜5+ビジュアル要素、余白を減らす
> 4. **エフェクト多用** — 爆発/キラキラ/ビックリ/丸/バツ等（※矢印は素材削除済み→シェイプ使用）
> 5. **テキストカラフル** — 1スライド3色以上
> 6. **赤ベースカラー** — パターン1と同じく赤系統で統一
> 7. **参考スライド** — `C:\KEI IWASAKI\参考スライド\パターン2\` の画像を基準
> 8. **ベースラインスクリプト** — gen-obs-p2-v5b.js を基準実装とする

## 素材管理

### パス指定方式（ハイブリッド）
- **デフォルトパス**: `C:\KEI IWASAKI\assets\{素材フォルダ名}\`（チャンネルプロファイル参照）
- ヒアリング時は「デフォルト(`suya_ai_movie`)でOK？」と確認
- 別パスが指定された場合はそちらを使用
- **普段は聞かないけど、変えたい時は指定できる**方式

### フォルダ構造
```
C:\KEI IWASAKI\assets\{チャンネル名}\
├── characters\     # 人物キャラ（PNG透過）表情バリエーション
├── effects\        # エフェクト（矢印、爆発、吹き出し等）
└── manifest.json   # メタデータ（自動生成）
```

タグ: `neutral`, `explain`, `shock`, `anger`, `happy`, `fear`

### manifest.json 自動生成ルール
- **タイミング**: ヒアリングで素材パスを受け取った直後に自動実行
- **処理**: フォルダ内の全画像をスキャン → ファイル名からmood/tagsを推定 → manifest.json生成
- **素材入れ替え時**: 放り込むだけでOK。次回ヒアリング時に自動再生成される
- **手動管理不要**

### エフェクト素材の用途ルール（厳守）
素材を「装飾」として無意味に散りばめるのはNG。必ず意味のある文脈でのみ使用。

| 素材 | 用途 | NG例 |
|------|------|------|
| 爆発 | 衝撃的事実の強調。1スライド最大1個 | 全スライドにバラ撒く |
| 吹き出し | 必ずテキストを入れる。セリフ・ツッコミ用 | 空の吹き出しを装飾で置く |
| キラキラ | 達成・成果・メリットの文脈のみ | 意味なく華やかにする |
| 上昇 | 数値が上がる文脈 | 装飾として適当に配置 |
| 下降 | 数値が下がる文脈 | 装飾として適当に配置 |
| ビックリ | 注意喚起・警告のみ | 全スライドに置く |
| はてな | 疑問提起・問いかけの文脈 | 装飾として適当に配置 |
| 丸（○） | 正解・推奨の比較表示 | 装飾として散りばめる |
| バツ（×） | 不正解・非推奨の比較表示 | 装飾として散りばめる |
| 矢印 | 素材から削除済み。pptxgenjsのシェイプ（ARROW_RIGHT等）を使用 | — |

> [!CAUTION]
> **原則：「このスライドのこの情報を強調するために置いた」と説明できること。説明できないなら置かない。**

### 新メンバー初回セットアップ
素材フォルダが未作成の場合、以下を案内：
1. `C:\KEI IWASAKI\assets\{チャンネル名}\` を作成
2. その中に `characters\` と `effects\` フォルダを作成
3. キャラPNG（透過）を `characters\` に、エフェクトPNGを `effects\` に入れる
4. ファイル名規則：**カテゴリ＋番号**（例：`笑顔_1.png`、`爆発_1.png`）

> [!TIP]
> ヒアリング時に素材フォルダが存在しなければ、上記手順を自動案内すること。

## 入力形式

### スプレッドシート（推奨）
`gws sheets` 経由で非公開でも読み込み可。

| 列名（例） | 必須 | 用途 |
|------------|------|------|
| 台本 / テキスト / script | **必須** | スライドのメインテキスト |
| スライド / イメージ / 補足 | 任意 | テンプレート選択のヒント |
| URL / リンク / 参考 | 任意 | スクショ引用・出典に使用 |
| 画像 / image | 任意 | 指定画像を貼り付け |
| セクション / 見出し | 任意 | 赤帯ヘッダーのタイトル |

### CSV / TSV / Markdown
ローカルファイルパスを指定。

## コード例

```javascript
const pptxgen = require("pptxgenjs");
const { parseScript } = require("./lib/script-parser");
const { renderSourceSlides } = require("./lib/pattern-source");

const pres = new pptxgen();
pres.layout = "LAYOUT_16x9";

const slides = parseScript(csvText, "source", { minSlides: 30, maxSlides: 50 });
renderSourceSlides(pres, slides);

await pres.writeFile({ fileName: "output/slides.pptx" });
```

## スライドタイプ（source用 — v4: 11種類）

> [!IMPORTANT]
> `text`（テキストのみ）と `screenshot`（スクショ引用）は**廃止**。
> スクショ素材の収集は `slide-source-collector` スキルに移管済み。

| type | 説明 | 判別キーワード | 映える用途 |
|------|------|---------------|------------|
| `table` | 比較表 | 比較、一覧、○×△ | 複数項目の横並び比較 |
| `chart` | グラフ（棒/折れ線/円） | 推移、チャート、割合 | 時系列データ、構成比 |
| `formula` | 数式・計算ステップ | 計算、利回り、シミュレーション | 投資計算、利回り算出 |
| `pyramid` | ピラミッド図 | レベル、Lv、段階 | 階層構造、レベル分け |
| `diagram` | 図解（フロー/因果/構造） | 仕組み、流れ、構造 | ビジネスモデル、因果関係 |
| `comparison` | 左右2列比較（A vs B） | vs、対比、違い | 2択の明確な対比 |
| `stats` | 大数字ドン＋補足テキスト | ％、倍、兆円 | インパクト数値の強調 |
| `timeline` | 横型タイムライン | 年表、歴史、経緯 | 時系列イベントの流れ |
| `checklist` | ✓/✗ 条件リスト | 条件、チェック、判定 | 該当/非該当の判定表 |
| `matrix` | 2×2マトリクス | リスク/リターン、分類 | 2軸での分類整理 |
| `cta` | CTA（LINE誘導等） | LINE、登録 | チャンネル登録・LINE登録 |

## フォントルール
- **最小フォント: 20pt**（それ以下は使わない。パターン1も2も同じ）
- 見出し: 28pt+
- キーワード: 36pt+
- ラベルボックス: 22pt
- 出典: 20pt
- 全テキスト太字、`FONTS.heading`使用
- Googleスライド互換のため `Noto Sans JP` + `Roboto`
